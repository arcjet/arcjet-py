# bindings/ — componentize-py generated reference code

This directory contains **guest-side** Python bindings generated by
[componentize-py](https://github.com/bytecodealliance/componentize-py). These
are included as a reference for the hand-written host-side bindings in
`arcjet_analyze/` and are **not intended to be shipped** in the final release.

The generated code is useful for:

- Understanding the WIT type shapes (records, variants, enums)
- Verifying function signatures against the canonical WIT definition
- Cross-referencing during development of the host-side `_types.py`

It cannot be used directly because:

- componentize-py generates *guest-side* bindings (for implementing a
  component), not *host-side* bindings (for calling one)
- The generated code uses `typing.Self` which requires Python 3.11+, while this
  project targets Python 3.10+

## Regenerating

```sh
uv tool install componentize-py
componentize-py -d ../filter.wit -w filter bindings ./bindings
```
